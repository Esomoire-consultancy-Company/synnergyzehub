# creators-common.org — SILK·AI Payments Adapter Alignment

## Final positioning

`creators-common.org` is the public creator-contribution and initiative-routing domain of the Commons ecosystem.

Its job is not merely to publish ideas. Its job is to let ideas, concepts, public-good proposals, location realities, civic studies, and creator contributions move into a governed execution path with identity, evidence, review, settlement, and audit.

The SILK·AI Dynamic Payments Adapter fits as the settlement layer behind that public `.org` surface. It classifies every financial event, routes it to the right rail, applies Warden evidence gates, computes auto-splits, and emits immutable RiverOS audit events.

## One-line definition

`creators-common.org` is the `.org` public initiative layer where creators submit, shape, publish, and route public-good concepts into governed programmes, with every accepted contribution linked to identity, evidence, payment, and audit.

## Ecosystem relationship

| Layer | Domain | Function |
|---|---|---|
| Creators Common | creators-common.org | Creates concepts, documents, designs, proposals, programme pages, and civic reality views |
| Believers Common | believerscommon.com | Backs, funds, protects, stewards, and gives continuity |
| BNR | internal/public governance label | Reviews, verifies, approves, pauses, audits |
| VSR | public browser/listing layer | Lists locations, programmes, NOCs, certificates, evidence records |
| Genesis Stack | infrastructure | Powers identity, runtime, evidence, settlement, and governance |
| SILK·AI Adapter | payment infrastructure | Routes payments, escrow, rewards, splits, fees, loyalty, maker disbursements |

## Public modules

### 1. Public Initiatives

Pages for water, textile recovery, orphan welfare, factory modernization, location development, election reality views, civic plans, and recovery programmes.

### 2. Creator Submission Desk

A submission route for creators, researchers, designers, engineers, students, writers, lawyers, and contributors to submit concepts or documents.

### 3. Programme Gazette

A public document register for each location, initiative, NOC, proposal, certificate, and public listing.

### 4. Reality View

A page that shows the present condition of a place, sector, or problem before proposing intervention.

### 5. Contribution Ledger

A public-facing summary of who contributed what, what was accepted, what was paid, what is pending, and what is under review.

### 6. Sponsor / Believer Backing

A route for Believers Common, institutions, businesses, families, and sponsors to fund or back specific public-good work.

### 7. VSR Public Listing Route

Accepted initiatives can become VSR listings, certificates, passports, NOCs, public records, or programme pages.

## Governance rule for the .org

Creators Common must not become an uncontrolled publishing wall.

Every public object must pass minimum acceptance:

```text
Identity
+ Permission
+ Context
+ Action
+ Evidence
+ BNR Review
+ Outcome
+ Record
= Accepted Public Object
```

## Commercial and governance boundary

Because this is a `.org`, the language should avoid looking like speculative investment or token promotion.

Use:

- contribution
- sponsorship
- programme backing
- public-good support
- creator reward
- escrow-backed work
- verified payment
- evidence-based release
- settlement record

Avoid:

- guaranteed returns
- investment promises
- token appreciation
- public token sale
- profit-sharing without legal structure
- unverified donation claims
